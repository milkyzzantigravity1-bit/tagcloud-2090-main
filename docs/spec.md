# Продуктовая спецификация

## Контекст

ГБОУ Школа №2090 нуждается в инструменте для проведения анонимных интерактивных опросов на классных часах, мероприятиях и педсоветах. Результаты должны визуализироваться в виде облака слов — наглядно, без числовых счётчиков.

## Роли

| Роль | Регистрация | Идентификация |
|---|---|---|
| **Создатель опроса** | **Email + пароль** (этап 6). Регистрация даёт доступ ко всем своим опросам через `/my`. Существующие опросы с email из ТЗ-этапа автоматически «усыновляются» при первой регистрации с этим email. | Cookie-сессия (httpOnly, 30 дней). Бэккомпат: старые URL `/s/CODE?t=<token>` всё ещё работают. |
| **Респондент** | Не нужна. Полностью анонимен. | Не идентифицируется. Только rate-limit по хэшу IP без сохранения. |

## User stories

### Создатель

- Как учитель, я хочу создать опрос с одним или несколькими вопросами, чтобы провести интерактив на классе.
- Хочу указать срок действия — после этого голосование закрывается, ничего нового никто не пришлёт.
- Хочу выбрать тип ответа (одно слово / несколько слов) для каждого вопроса.
- Хочу выбрать, влияет ли регистр на агрегацию (`Россия` и `россия` — одно слово или разные).
- Хочу настроить цвет облака: чёрно-белое, случайные яркие цвета, или своя палитра.
- После создания хочу получить **код комнаты** (короткий, чтобы продиктовать), **ссылку** и **QR-код** (для проектора).
- В реальном времени хочу видеть, как растёт облако.
- После окончания опроса хочу получить на email **картинку облака** + **CSV-таблицу** "слово, количество".

### Респондент

- Я хочу зайти по короткой ссылке или отсканировать QR — без логинов и без капчи.
- Я хочу прочитать вопрос(ы) и ввести ответ(ы).
- Если вопрос помечен как "одно слово" — поле не должно мне разрешить вписать пробел.
- Я не хочу видеть свой ответ в общем облаке именно как мой — только как анонимный вклад.

## Функциональные требования

### F1. Создание опроса

- Создатель указывает: **email** (обязателен), **название** (опционально), **срок** (datetime, в пределах 1ч–30 дней), **caseSensitive** (boolean, default `false`), **colorScheme** (`mono` | `random` | `custom`).
- Если `colorScheme=custom` — указывает массив hex-цветов (1–10 значений).
- Создатель добавляет **1–50 вопросов**. Для каждого: **текст** (≤500 символов), **тип** (`single` | `multi`).
- На сервере создаётся: запись `surveys`, записи `questions[]`, генерируется уникальный 6-символьный `code` из алфавита `ABCDEFGHJKMNPQRSTUVWXYZ23456789` (без 0/O/1/I/l).
- В ответе клиент получает: `code`, `url` (`/r/<code>`), `qrPngBase64`, `expiresAt`, `creatorToken` (UUID для дашборда).
- Дашборд создателя: `/s/<code>?t=<creatorToken>`.

### F2. Голосование

- Респондент открывает `/r/<code>` — видит список вопросов опроса. Если опрос истёк или не найден — экран "Опрос завершён / не найден".
- Для каждого вопроса:
  - `single` — `<input type="text">` с JS-обработчиком: `keydown` пробела → `preventDefault()`. Подсказка под полем "одно слово".
  - `multi` — несколько слов через запятую/Enter (UX-выбор финализируется в этапе 3, default — отдельный input на каждое слово с кнопкой "+").
- Лимиты: длина слова **≤50 символов**, в `multi` **≤20 слов** на вопрос.
- Сервер валидирует повторно (клиентскую валидацию обходят). При нарушении — 400 с понятной русской ошибкой.
- После отправки — confirmation-экран "Спасибо!". Опционально показ итогового облака (если создатель разрешил — флаг на этапе 4).
- Защита от двойного голосования:
  - localStorage-токен `voted:<code>` — best-effort.
  - Redis-ключ `voted:<sha256(ip+salt)>:<code>` — TTL до `expiresAt`. Salt ротируется каждые 24 часа.

### F3. Realtime-обновление облака

- На дашборде создателя облако обновляется автоматически в течение нескольких секунд после голоса.
- Используется WebSocket: клиент подписывается на `/ws/<code>?t=<creatorToken>`. Сервер шлёт `snapshot` сразу + `delta` каждые 2-3 сек.
- Облако строится `wordcloud2.js`: размер шрифта пропорционален частоте (логарифмическая шкала), популярные слова — ближе к центру.
- **Числовых счётчиков рядом со словами нет** — только размер.

### F4. Цветовые схемы облака

- `mono` — все слова в фирменном `--c-navy` (`#0E2A5C`) на белом фоне.
- `random` — случайный цвет из встроенного бренд-набора (`navy`, `blue`, `gold`) для каждого слова.
- `custom` — случайный цвет из палитры, указанной создателем (1–10 hex-значений).

### F5. Истечение и email-рассылка

- Каждую минуту cron сканирует `surveys` с `status='active' AND expires_at < now()`.
- Для каждого:
  1. Агрегирует ответы: `SELECT word_norm, count(*) FROM responses WHERE question_id IN (...) GROUP BY word_norm`.
  2. Для каждого вопроса рендерит **PNG облака** (1200×800) с цветовой схемой опроса. Использует `node-canvas` + `wordcloud2`.
  3. Собирает **CSV**: колонки `question, word, count` (UTF-8 + BOM для Excel).
  4. Шлёт письмо на `creator_email` через **Yandex SMTP** (`smtp.yandex.ru:465`, SSL). Тема: «Результаты опроса "{title или code}"». В теле — компактная HTML-вёрстка с инлайн-облаками. Аттач: PNG-ы и CSV.
  5. `UPDATE status='sent'`. При ошибке — `status='failed'`, повтор через 1 час, максимум 3 попытки.
- Записи `responses` НЕ удаляются. Создатель может перезапросить CSV вручную через `/api/surveys/:code/export.csv`.

### F6. Экспорт CSV

- GET `/api/surveys/:code/export.csv?t=<creatorToken>` доступен в любой момент после создания (даже если опрос ещё активен).
- Формат: `question_text,word,count\n` с UTF-8 BOM. Сортировка: по вопросам, внутри — по убыванию count.

### F7. Брендирование

- Логотип ГБОУ Школа №2090 в шапке всех страниц (`static/logo2090.png`).
- Цветовая палитра — производная от логотипа:
  - `--c-navy: #0E2A5C` (основной)
  - `--c-blue: #2D9FDA` (акцент)
  - `--c-gold: #E0B73A` (акцент 2)
- Шрифт `Inter` (бесплатный, кириллица).
- Минимализм: одна колонка ≤880 px, много white-space, без лишних теней и градиентов.

## Нефункциональные требования

| Требование | Решение |
|---|---|
| Бесплатность разработки и поддержки | Open-source стек, бесплатные планы (Yandex SMTP free 500/день, DuckDNS free) |
| Домашний сервер | Docker Compose, без managed сервисов |
| 1000 одновременных | Один процесс Node + Postgres + Redis. Узкое место — upload-канал; митигация — throttle WS до 2-3 сек |
| Анонимность | Не сохраняется IP/fingerprint/UA. Rate-limit — sha256(ip+salt) с TTL 10 мин |
| Минимализм UI | См. F7 |
| Брендирование 2090 | См. F7 |
| Бэкапы | `pg_dump` daily + (опц.) sync на Backblaze B2 free 10 GB (этап 7) |

## Решённые вопросы

- **Email-провайдер**: Yandex SMTP (не Brevo). Лимит 500/день — достаточно, т.к. письмо отправляется 1 раз на ОПРОС.
- **Frontend стек**: SvelteKit (выбран за меньший bundle).
- **Длина кода**: 6 символов из 31-символьного алфавита (≈900 млн комбинаций — коллизии практически невозможны).
- **Защита от двойного голосования**: localStorage + Redis hash(IP+salt). Гибрид.
- **Лимит вопросов на опрос**: 50.
- **Лимит длины слова**: 50 символов.
- **Лимит слов в multi**: 20 на вопрос.
- **Срок опроса**: от 1 часа до 30 дней.

## Открытые вопросы

- Можно ли респонденту видеть итоговое облако сразу после голосования? **Решение для этапа 4**: флаг на опросе `showResultsToVoters: boolean`, default `false`.
- Нужен ли дашборду экспорт PNG облака до истечения опроса? **На этапе 4 — кнопка "скачать снимок облака"** (используется тот же node-canvas рендер, что для email).
- Нужно ли уведомлять создателя по email о приближении срока истечения? **Из ТЗ не следует — не делаем**.
